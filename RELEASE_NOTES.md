# Release Notes

[ListKit](https://github.com/danielsaidi/ListKit) will honor semantic versioning after version 1.0.

* Deprecations can happen at any time.
* Deprecations should only be removed in `major` updates.
* Breaking changes should only occur in `major` updates.
* Breaking changes *can* occur in non-major updates, if the alternative is worse.

Until then, breaking changes can also happen in `minor` updates.



## 0.1

This is the first beta release of ListKit.
